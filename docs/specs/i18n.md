I18n covers translation file discovery, language detection, language switching, translation lookup, plural lookup, literal key coverage, and translation report tooling. Runtime source lives mainly in `Commons/I18n.qml`; implementation notes belong in [docs/wiki/systems/i18n.md](../wiki/systems/i18n.md).

## What it must do

### Language discovery and detection

- [x] Language scanning logs translation directory scans and starts the scanner process.
- [x] Directory listing parsing falls back to English and detects language when output is empty.
- [x] Directory listing parsing reads newline-separated entries, accepts only JSON translation files, and rejects malformed language codes.
- [x] Directory listing parsing keeps English first when English is available.
- [x] Directory listing parsing logs parse failures, falls back to English, and detects language after failure.
- [x] Language detection stops and logs a warning when no languages are available.
- [x] Language detection inspects Qt preferred UI languages.
- [x] Language detection prefers exact locale matches before short language-code matches.
- [x] Language detection falls back to English or the first available language.
- [x] Language detection prefers a configured language when settings provide one.
- [x] Language detection applies the detected locale when no configured language is valid.

### Loading and lookup

- [x] Setting a language defaults the full locale to the language code when no full locale is supplied.
- [x] Setting a language reloads only changed, available languages.
- [x] Setting a language updates language code, full locale code, and Qt locale together.
- [x] Setting a language emits the language-changed signal and reloads translations after a change.
- [x] Setting a language warns when the requested language is unavailable.
- [x] Translation loading no-ops until a language is selected.
- [x] Translation loading points the FileView at the selected translation file and clears loaded state.
- [x] Translation loading loads English fallback translations for non-English languages when English is available.
- [x] Translation existence checks return false until translations are loaded.
- [x] Translation existence and lookup helpers type required key inputs, and plural lookup types the count input.
- [x] Translation existence checks walk nested keys and accept only leaf strings.
- [x] Key enumeration defaults to active translations, preserves nested key paths, recurses into nested objects, and returns string leaves.
- [x] Reloading logs and delegates to translation loading.

### Translation and plural lookup

- [x] Translation lookup defaults interpolations to an object.
- [x] Translation lookup returns the key before translations are loaded.
- [x] Translation lookup supports nested translation keys and searches active translations first.
- [x] Translation lookup falls back to English when a non-English active language misses a key.
- [x] Translation lookup makes missing fallback keys visible.
- [x] Translation lookup marks fallback strings as untranslated.
- [x] Translation lookup rejects non-string translation values.
- [x] Translation lookup replaces all matching interpolation placeholders.
- [x] Plural lookup defaults singular and plural fallback text.
- [x] Plural lookup chooses singular or plural keys from count.
- [x] Plural lookup injects count, preserves caller interpolations, and delegates final lookup through translation lookup.

### Assets and tooling

- [x] Literal `I18n.tr("...")` calls in QML must point to string leaves in the English translation file, excluding dynamic key prefixes.
- [x] Literal translation reference coverage must remain broad enough to catch project-wide missing keys.
- [x] The `i18n-json` development report identifies the target translation file, total reference keys, target keys, completion percentage, missing keys, and extra keys.

## Implementation inventory

- `Commons/I18n.qml` - translation language discovery, language selection, translation loading, key lookup, interpolation, plural lookup, and fallback handling.
- `Assets/Translations/en.json` - English reference translations used by runtime fallback and literal-key validation.
- `Assets/Translations/*.json` - localized translation files discovered by I18n and audited by development tooling.
- `Bin/dev/i18n-json.sh` - translation completeness report for a target locale against the reference file.
